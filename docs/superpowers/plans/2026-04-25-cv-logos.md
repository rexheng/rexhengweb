# CV Logos Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add self-hosted SVG organisation logos to CV entry-meta columns and project card title rows; remove the Attic project card.

**Architecture:** Single-file HTML page (`cv/index.html`). SVGs stored in `cv/logos/`. No bundler, no JS — all changes are HTML attribute additions, CSS additions, and SVG file downloads. Each SVG root must carry explicit `width`/`height` attributes for CLS-safe layout reservation.

**Tech Stack:** Static HTML, CSS, SVG files

**Spec:** `docs/superpowers/specs/2026-04-25-cv-logos-design.md`

## Chunk 1: CSS + Attic removal

### Task 1: Add logo CSS to cv/index.html

**Files:**
- Modify: `cv/index.html` (inside `<style>` block, after the `/* ─── Print ─── */` block at line ~493)

- [ ] **Step 1: Add print filter reset**

Inside the existing `@media print { ... }` block (around line 494–519), add the following line before the closing `}` brace of the print block:

```css
      .entry-logo, .pc-logo { filter: none; }
```

Do this step first so the subsequent insertion at `</style>` doesn't shift the print block's line numbers.

- [ ] **Step 2: Add entry-logo and pc-logo CSS rules**

Insert the following immediately before the closing `</style>` tag (after the `@media print` block):

```css
    /* ─── Entry logos ─── */
    .entry-logo {
      display: block;
      height: 32px;
      width: auto;
      margin-top: 12px;
      filter: grayscale(1) opacity(0.55);
      transition: filter 240ms ease;
    }
    .entry:hover .entry-logo { filter: grayscale(0) opacity(1); }

    /* ─── Project card logos ─── */
    .pc-logo {
      display: block;
      height: 18px;
      width: auto;
      flex-shrink: 0;
      align-self: center;
      filter: grayscale(1) opacity(0.5);
      transition: filter 220ms ease;
    }
    .project-card:hover .pc-logo { filter: grayscale(0) opacity(1); }
```

- [ ] **Step 3: Verify CSS is syntactically correct**

Open `cv/index.html` in a browser (or run `npx html-validate cv/index.html` if available). No parse errors expected.

- [ ] **Step 4: Commit**

```bash
git add cv/index.html
git commit -m "feat(cv): add entry-logo and pc-logo CSS rules"
```
