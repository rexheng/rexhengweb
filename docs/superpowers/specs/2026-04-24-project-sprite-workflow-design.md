# Project Sprite Workflow — Phase A

**Date:** 2026-04-24
**Scope:** Refactor `src/projects.js` into a modular catalog structure, port Amogus as-is into the new structure, then rebuild Amogus and Republic (Doric pillar) properly using a reference-first 7-step workflow.
**Out of scope:** Ralph loop, remaining projects from `projects/index.html`. Those get a separate Phase-B spec once Phase A is verified.

---

## 1. Why

Today's `src/projects.js` mixes the slot-pool system, the mesh builders, the ability physics, and the project registry in one 529-line file. The Amogus mesh was written from prose ("rounded torso, cyan visor") with no reference image, no proportion sheet, and no side-by-side silhouette check — so it reads as a generic red bean rather than an Amogus crewmate.

Root cause: the workflow was **prose → Three.js**. That's lossy. Adding more projects under the same workflow compounds the loss (eight ambiguous red beans instead of one).

The new workflow makes the reference image and the proportion sheet **filesystem-enforced artifacts** that the build code imports from. You literally cannot write a builder without a proportions file sitting next to it.

---

## 2. The 7-step workflow (per project)

Every sprite follows this, in order. No step skipped.

1. **Brief** — `catalog/<id>/BRIEF.md`: one-liner, link to the project card on `projects/index.html`, and the sprite's intended role in the scene (prop / character / vehicle / landmark).
2. **Refs** — fetch 3–5 reference images to `catalog/<id>/refs/` (front, side, detail, material/colour swatch). Write `REFS.md` listing: source URL for each image, what it shows, the colour hexes pulled from it, and (where relevant) the canonical dimensions the image documents.
3. **Decompose** — a comment block at the top of `proportions.js` breaks the silhouette into primitive blocks: "base = stepped cylinder 1.25×D, shaft = fluted cylinder 1.0×D × 6×D, capital = square abacus 1.35×D × 0.25×D". Prose + ASCII if helpful.
4. **Proportions** — `proportions.js` exports numeric constants, every dimension expressed as a ratio of the **base unit** `D = 0.26` (the slot's collision capsule radius). Colour hexes exported too. **No THREE imports, no logic, just `export const`.**
5. **Build** — `build.js` exports `buildMesh({ THREE, materials, primitives, proportions })`. Pure function. Reads every number and colour from `proportions.js` via its argument — no inline magic numbers.
6. **Visual check** — `npm run render:<id>` renders the mesh headless at a fixed camera and saves `refs/render.png`. Eyeball next to `refs/front.png`. Iterate `proportions.js` constants until the silhouette matches. Commit when the two PNGs sit side-by-side satisfactorily.
7. **Wire** — `catalog/<id>/index.js` exports the full project def (`{ id, label, title, subtitle, meta, description, accent, links, ability, buildMesh }`). Run `npm run regen-slots` to regenerate the MJCF slot block from the registry. Open the site, spawn via the "Add Project" button, test ability + grab + physics.

Steps 2 → 4 produce artifacts that step 5 **imports from**. The link is enforced by the filesystem and the import graph, not by discipline.

---

**Note on Step 4 vs Step 6 ("port as-is" vs "rebuild properly"):** the as-is port in §8 step 4 is explicitly **non-conforming** to the catalog contract — it exists only to keep the site runnable while the structural refactor lands. The contract from §4.1 ("no inline magic numbers in `build.js`") is **suspended** for the file produced in step 4 and **enforced** starting in step 6 when Amogus is rebuilt against `proportions.js`. Acceptance criterion §9 is checked at step 6+, not step 4.

## 3. Module structure

```
src/projects/
  index.js                     ← auto-discovers catalog/*/index.js, exports PROJECTS
  system.js                    ← ProjectSystem class (slot pool, labels, cards)
  abilities/
    index.js                   ← { stab, lunge, spin, topple, … } keyed by string
    impulse.js                 ← shared helpers: threeDirToMj(), writeQvel(), findBody()
  builders/
    primitives.js              ← tictac(), flutedColumn(), torusBand(), steppedCylinder()
    materials.js               ← makeMaterials(THREE) → { matteRed, marbleWhite, … }
  catalog/
    amogus/
      index.js                 ← project def, imports buildMesh + ability name
      build.js                 ← buildMesh() — pure, dependency-injected
      proportions.js           ← export const SIZES, COLOURS
      BRIEF.md
      REFS.md
      refs/
        front.png  side.png  detail.png  swatch.png  render.png
    republic/
      (same layout)
assets/scenes/
  humanoid.xml                 ← slot block regenerated by script, never hand-edited
scripts/
  regen-slots.mjs              ← reads catalog/, rewrites the <!-- SLOT_BLOCK --> region
  render-sprite.mjs            ← headless three.js render for step 6
```

**Why this shape, restated for bug-resistance:**

- One folder per project = blast radius of any edit is one folder. Breaking Republic cannot break Amogus.
- `proportions.js` has zero imports and zero logic, so the spec values **are** the runtime values. Drift impossible.
- `refs/` lives next to the code that consumed it. Orphaned images are visually obvious in the tree.
- Shared helpers in `abilities/impulse.js` and `builders/primitives.js` = one place to fix the THREE↔MJ swizzle, one place to fix capsule construction.
- Slot block in the MJCF is **generated**, so adding a project is never an XML edit. Slot count is fixed at `SLOT_COUNT = 8` for Phase A regardless of `PROJECTS.length` (so the pool is always large enough for hand-spawned duplicates and there's headroom for Phase B).

**ProjectSystem responsibilities (full list — all rehomed to `system.js`):** slot pool indexing, slot claim/park lifecycle, label DOM host + per-frame screen-projection, info card DOM, click-to-spawn binding, **collision toggling (`contype`/`conaffinity` flip on claim and park — currently `_setSlotCollision`)**, **body lookup (`_findBodyByName`)**, **grabbable rebinding (`app._rebuildGrabbables` call after mesh attach)**, **`bodyID` tagging on every mesh descendant for the grabber**. Nothing in this list moves to `main.js`, `catalog/`, `builders/`, or `abilities/`.

**Validation at glob-load time:** `src/projects/index.js` validates each catalog entry. It asserts (a) `proportions.D`, `proportions.SIZES`, `proportions.COLOURS` exist and every numeric value in `SIZES` is finite; (b) `def.ability` is either `null` or a key in `ABILITIES`; (c) required string fields (`id`, `label`, `title`) are present. A failing project is **excluded from `PROJECTS` with a single `console.error` line**, never silently broken — the rest of the catalog still loads, the site doesn't crash.

## 4. Concrete file contracts

### 4.1 `proportions.js` (template)

```js
// Base unit D is the slot collision capsule radius. Every dimension below
// is expressed as a multiple of D so the sprite scales with the slot.
export const D = 0.26;

export const SIZES = {
  torsoRadius:   1.00 * D,
  torsoHeight:   2.40 * D,   // flat-bottomed capsule → Tic-Tac silhouette
  visorWidth:    1.20 * D,
  visorHeight:   0.45 * D,
  visorZOffset:  0.95 * D,   // height from torso centre
  // …every dimension the build uses, named
};

export const COLOURS = {
  body:        "#d94a52",    // pulled from refs/swatch.png pixel (124, 86)
  bodyShadow:  "#7a2228",
  visorGlass:  "#7ad5e0",
  visorFrame:  "#2a3844",
};
```

**Rule:** if `build.js` types a numeric literal or a colour hex, that's a bug. Code review rejects it.

### 4.2 `build.js` (template)

```js
// Pure builder. All dependencies injected — no imports of THREE, materials,
// or primitives. Lets the render script and the live app share one implementation.
export function buildMesh({ THREE, materials, primitives, proportions }) {
  const { SIZES, COLOURS } = proportions;
  const group = new THREE.Group();
  group.name = "amogus_mesh";

  const mats = materials.forProject({ body: COLOURS.body, shadow: COLOURS.bodyShadow, … });

  group.add(primitives.tictac({
    radius: SIZES.torsoRadius,
    height: SIZES.torsoHeight,
    material: mats.body,
  }));
  // …
  return group;
}
```

### 4.3 `catalog/<id>/index.js` (template)

```js
import { buildMesh } from "./build.js";
import * as proportions from "./proportions.js";

export default {
  id: "amogus",
  label: "Amogus",
  title: "Amogus — 2nd Cursor Hackathon",
  subtitle: "2024 · a sus little guy",
  meta: "three.js · procedural",
  accent: "#e25d63",
  description: "…",
  links: [{ label: "Source", href: "#" }],
  abilityLabel: "Stab!",
  ability: "stab",             // string key, resolved via abilities/index.js
  buildMesh: (deps) => buildMesh({ ...deps, proportions }),
};
```

Every project def is this shape. No exceptions. `src/projects/index.js` globs `catalog/*/index.js` at build time (Vite) and assembles `PROJECTS`.

### 4.4 `abilities/index.js`

```js
import { stab } from "./stab.js";
import { topple } from "./topple.js";      // Republic's ability
export const ABILITIES = { stab, topple };
```

`ProjectSystem._fireAbility` looks up by string: `ABILITIES[def.ability]({ app, slot })`.

### 4.5 `scripts/regen-slots.mjs`

Reads `src/projects/catalog/*/index.js` (only to confirm the catalog loads cleanly), then rewrites `assets/scenes/humanoid.xml` between `<!-- SLOT_BLOCK_BEGIN -->` and `<!-- SLOT_BLOCK_END -->` markers. Always emits exactly `SLOT_COUNT = 8` slots. Idempotent. Fails loudly if markers absent.

**Exact emitted template** (per slot, indented with 4 spaces, one slot per line, attributes in this order):

```xml
    <body name="project_slot_${i}" pos="0 0 ${(-60 - i * 0.5).toFixed(1)}"><freejoint/><geom name="project_slot_geom_${i}" type="capsule" fromto="0 0 -0.22 0 0 0.22" size="0.26" rgba="0.85 0.18 0.20 1" mass="2.5" friction="1.0 0.05 0.002" contype="0" conaffinity="0"/></body>
```

`i` ranges `0..7`. The block is preceded by the existing explanatory comment (the one at lines 185-193 of the current `humanoid.xml` describing why `contype/conaffinity=0` matters) — the regen script preserves that comment verbatim inside the marker region. **Acceptance:** running the script against the current `PROJECTS = [amogus]` produces a slot block byte-for-byte identical to the current hand-written one.

### 4.6 `scripts/render-sprite.mjs`

CLI: `node scripts/render-sprite.mjs amogus`. Uses `three` + `node-canvas` (or `playwright` fallback) to render the mesh against a neutral grey background at a fixed camera (azimuth 30°, elevation 15°, distance 3×`SIZES.torsoHeight`), saves to `catalog/amogus/refs/render.png`. Runs in CI as a smoke test — if the import graph breaks, CI fails before the site does.

## 6. Republic — Doric pillar

**Brief:** marble Doric column, fallen-in-the-forum look; topples when its ability fires.

**Refs:** fetch three from Wikimedia — a standing Doric column (full silhouette), a capital close-up, a shaft close-up showing fluting. Source URLs recorded in `REFS.md`.

**Silhouette decomposition:**

```
┌──────┐     abacus      — square slab, 1.35×D × 0.25×D
 ╲────╱       echinus      — downward-flared torus/frustum, 1.20×D × 0.20×D
 │    │       necking      — short cylinder, 0.95×D × 0.15×D
 │≡≡≡≡│       shaft        — fluted cylinder, 1.00×D × 5.00×D, ~20 flutes
 │    │       (entasis: top radius = 0.85 × bottom radius, subtle taper)
 ╲────╱       base torus   — torus band, 1.15×D × 0.15×D
└──────┘     plinth       — short wide cylinder, 1.30×D × 0.20×D
```

Total height ≈ 6.05×D ≈ 1.57m. The sprite is **tall**, so it visibly towers over the 1m humanoid — intentional, reads as a landmark.

**Proportions constants** (in `catalog/republic/proportions.js`):

```js
export const D = 0.26;
export const SIZES = {
  plinthR: 1.30*D,    plinthH: 0.20*D,
  baseR:   1.15*D,    baseH:   0.15*D,
  shaftRBottom: 1.00*D,  shaftRTop: 0.85*D,  shaftH: 5.00*D,  flutes: 20,
  neckingR: 0.95*D,   neckingH: 0.15*D,
  echinusRTop: 1.20*D, echinusRBottom: 0.98*D, echinusH: 0.20*D,
  abacusW: 1.35*D,    abacusH: 0.25*D,
};
export const COLOURS = {
  marble:       "#e8e2d5",   // weathered warm white
  marbleShadow: "#b8aa90",
  moss:         "#6b7a3f",   // thin moss band at base, optional
};
```

**Build notes:**

- Shaft fluting = 20 vertical grooves cut as a `LatheGeometry` with a scalloped profile, OR a single smooth `CylinderGeometry` with a normal-map fake if lathe perf is bad. Start with lathe, fall back only if slow.
- Entasis (subtle taper + bulge) comes free from the lathe profile.
- Material: `MeshPhysicalMaterial` with `roughness: 0.85`, `clearcoat: 0`, a touch of `ao` from a baked texture if we want weathering. Start plain.

**Ability:** `topple` — applies a rotational impulse perpendicular to the "forward" axis, plus a small horizontal nudge. The pillar falls over like a felled tree. New ability, lives in `abilities/topple.js`, reusable for any tall project later (lighthouses, statues, …).

**Accent colour for card/label:** `#c9b58a` (warm sandstone).

**Silhouette acceptance table** (mirror of §5; checked at step 6):

| Feature | Target (from refs) | Pass condition |
|---|---|---|
| Overall proportion | Tall and thin, ~6×D total height vs ~1.3×D max width | Side-by-side render: column visibly ~4-5× taller than wide |
| Capital | Square abacus on top of flared echinus (mushroom profile) | Abacus is square (not round), echinus visibly flares out from shaft |
| Shaft | Visible vertical fluting + subtle entasis | At least 12 of 20 flutes readable from front render; top noticeably narrower than bottom |
| Base | Torus band sitting on a wider square-ish plinth | Two distinct horizontal bands at the base, plinth wider than base torus |
| Material | Weathered warm marble, not chalk-white, not grey | Pixel sample of shaft mid-render falls in `#d8d0bf`–`#eee6d8` range |

## 7. Slot block generation

Current MJCF has 8 hand-written slot bodies. Replace the block with marker comments:

```xml
<!-- SLOT_BLOCK_BEGIN — regenerated by scripts/regen-slots.mjs, do not edit by hand -->
… 8 bodies, one per PROJECTS entry, each with contype=0 conaffinity=0 …
<!-- SLOT_BLOCK_END -->
```

`regen-slots.mjs` reads `PROJECTS.length`, emits `max(PROJECTS.length, 8)` slots, preserves the `contype/conaffinity=0` quirk (critical for avoiding NaN at load time — currently documented in the XML comment, which the regen script must preserve).

## 8. Refactor plan (Phase A execution order)

The plan phase (next skill) will break this into ordered tasks. At spec level, the sequence is:

1. **Create module skeleton** — new dirs, empty files, no behaviour change yet. `src/projects.js` untouched. Site still works.
2. **Extract `ProjectSystem` into `src/projects/system.js`.** `src/projects.js` becomes a one-line re-export: `export { ProjectSystem } from "./projects/system.js"; export { PROJECTS } from "./projects/index.js";` (where `index.js` is still empty/stub). `src/main.js` import line **unchanged** at this step. Site still works.
3. **Extract helpers** into `abilities/impulse.js`, `builders/primitives.js`, `builders/materials.js`. `system.js` updated to import from them. Site still works.
4. **Port existing Amogus as-is** into `catalog/amogus/build.js` + `catalog/amogus/index.js`. **Catalog contract suspended for this step** — magic numbers preserved, no `proportions.js` required yet. `src/projects/index.js` is wired up to glob the catalog and export `PROJECTS`. The shim in `src/projects.js` now re-exports the real registry. Site still spawns Amogus identically.
5. **Add MJCF markers + regen script.** Wrap the existing slot block with `<!-- SLOT_BLOCK_BEGIN -->` / `<!-- SLOT_BLOCK_END -->`. Implement `scripts/regen-slots.mjs` per §4.5. Run it; assert byte-identical output. One commit.
6. **Rebuild Amogus properly** — the workflow's first worked example. Add `BRIEF.md`, `REFS.md`, `refs/`, `proportions.js`. Rewrite `build.js` to read from proportions (now contract-conforming). Run `render:amogus`. Iterate until the §5 silhouette table passes. Commit.
7. **Build Republic** from scratch with the full workflow. Refs, proportions, build, render, iterate against the §6 silhouette table. New `abilities/topple.js`. Run `regen-slots` (still emits 8 — slot count is fixed, only registry changes). Commit.
8. **Update `src/main.js`** to import `ProjectSystem` from `./projects/system.js` directly. **Delete `src/projects.js`** (the shim). Final commit.

Each step commits independently. A regression bisects to one step. **The site is runnable after every commit** — the shim in steps 2-7 is what guarantees this.

## 9. Acceptance criteria

Phase A is done when:

- [ ] `src/projects.js` no longer exists; `src/projects/` module is in its place.
- [ ] `npm run render:amogus` produces a PNG that, next to `refs/front.png`, reads as the same character (flat-bottom body, wide wraparound visor, stub legs wide apart).
- [ ] `npm run render:republic` produces a PNG that reads unambiguously as a Doric column from the forum.
- [ ] Both sprites spawn via "Add Project", can be grabbed, collide with the humanoid, and fire their abilities without errors.
- [ ] `npm run regen-slots` is idempotent and the MJCF slot block is byte-identical across two consecutive runs.
- [ ] Adding a hypothetical third project requires **only** creating `catalog/<newid>/` and running `regen-slots`. No edit to `system.js`, `main.js`, or `humanoid.xml` outside the marker region.
- [ ] The `stab` ability behaves identically to the pre-refactor version (one-shot impulse, same speed + spin).

## 10. Known risks and mitigations

| Risk | Mitigation |
|---|---|
| Lathe geometry for fluted shaft is slow on low-end GPUs | Measure in step 6. Fall back to smooth cylinder + normal map if > 2ms/frame delta. |
| `node-canvas` headless render is flaky on Windows | Fall back to Playwright-driven render of a minimal HTML harness. Decide at plan time. |
| Vite glob import of `catalog/*/index.js` might not cover dynamic add during dev | Use `import.meta.glob('./catalog/*/index.js', { eager: true })`. Well-supported, documented. |
| Regen script corrupts MJCF if markers drift | Script hashes the pre-write file, writes to a tmp, diff-check, atomic rename. Fails loudly on missing markers. |
| THREE→MJ swizzle bug in shared `impulse.js` breaks every ability at once | Unit test the helper: given a known THREE direction, assert the MJ output. One test, catches the class. |
| Ralph (Phase B) blast radius | Out of scope here, but Phase A's folder-per-project shape is the mitigation. |
| `proportions.js` missing or malformed → `import * as proportions` returns `{}`, builder crashes with NaN geometry | Glob-load validator in `src/projects/index.js` excludes the project with a `console.error`; site continues to load with the remaining catalog. |
| Project def omits `ability` → click on action button calls `ABILITIES[undefined]({...})` and throws | Validator rejects defs whose `ability` is neither `null` nor a key in `ABILITIES`. Card UI's existing `def.ability` truthy guard (current `projects.js:507`) is preserved. |
| Mid-refactor site breakage between commits 2-7 | `src/projects.js` becomes a re-export shim in step 2 and stays one until step 8. `main.js` import line is unchanged until step 8. Bisect-safe. |
| `LatheGeometry` UV/normal seams visible on the fluted shaft | Acceptance is the silhouette table in §6, not pixel-perfect surface. If seams are objectionable, fall back to the cylinder + normal-map plan from row 1. |
